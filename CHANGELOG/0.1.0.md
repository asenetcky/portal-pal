# 0.1.0

Initial release of Portal Pal — a deployable LLM backend for querying open data portals via natural language.

## Features

- **LangGraph Agent** — LangGraph-powered agent with LangSmith observability wired in for tracing production requests.
- **FastAPI backend** — async HTTP service with `/chat`, `/health`, `/metrics`, and `/cache/stats` endpoints.
- **API key authentication** — `/chat`, `/metrics`, and `/cache/stats` gated behind `X-API-Key` header; `/health` left open for health check probes. Key configured via `PORTAL_PAL_API_KEY` env var.
- **PII protection** — `PIIDetector` masks SSNs, credit cards, and phone numbers before input reaches the LLM. `@traceable` decorators removed from pre-masking functions to prevent raw user input from leaking to LangSmith.
- **Monorepo structure** — source split into `apps/backend/` (FastAPI + RAG) and `apps/frontend/` (placeholder) workspaces managed by uv.
- **Docker support** — `Dockerfile` and `docker-compose.yml` for local orchestration.
- **Render deployment** — `render.yml` config for one-click deploy to Render.
- **Release tooling** — `scripts/release.py` and `just release` recipe to tag, push, and publish GitHub releases from changelog files.

## Changes from scaffold

- Removed PyPI publishing infrastructure; project is an application, not a library.
- Renamed from `open-data-portal-pal` to `portal-pal`.
- Restructured source from `src/open_data_portal_pal/` into the monorepo layout.
- CI hardened for private repo: per-job `contents: read` permissions, required env vars injected for pydantic-settings validation, coverage source path updated post-reorg.
